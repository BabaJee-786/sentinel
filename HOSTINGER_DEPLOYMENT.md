## Database Setup

### Step 1: Create MySQL Database on Hostinger

1. Log in to Hostinger hPanel
2. Navigate to: **Databases** > **MySQL Databases**
3. Create a new database:
   - **Database name**: `sentinel_prod` (or your preference)
   - **Database user**: `sentinel_user`
   - **Password**: Generate strong password (save for .env file)
   - **Privileges**: Select all privileges

### Step 2: Run Database Migrations

**Option A: Using phpMyAdmin (Recommended for first-time setup)**

1. Log in to phpMyAdmin
2. Select your Sentinel database
3. Click "Import" tab
4. Upload `backend/database/migrations.sql`
5. Upload `backend/database/migrations_v2.sql` (new features)
6. Click "Go"

**Option B: Using SSH (if available)**

```bash
# SSH into Hostinger
ssh your-hostinger-user@your-hostinger-server.com

# Navigate to project directory
cd public_html/sentinel

# Run migrations (MySQL client required)
mysql -u sentinel_user -p sentinel_prod < backend/database/migrations.sql
mysql -u sentinel_user -p sentinel_prod < backend/database/migrations_v2.sql

# Verify tables were created
mysql -u sentinel_user -p sentinel_prod -e "SHOW TABLES;"
```

### Step 3: Verify Database Structure

In phpMyAdmin, verify these tables exist:
- `devices` (should have `status` column)
- `domains` (should have `status` ENUM with 'live', 'paused' values)
- `device_domain_permissions` (NEW)
- `settings` (NEW)
- `email_logs` (NEW)
- `audit_logs` (NEW)

### Upload Process

**Using Hostinger File Manager (Easiest):**

1. Log in to Hostinger hPanel
2. Go to **File Manager**
3. Navigate to `public_html/sentinel/`
4. Upload files using drag-and-drop or file upload

**Using FTP/SFTP (Recommended for large uploads):**

```bash
# Using FileZilla or similar FTP client

1. Connect to Hostinger SFTP
   - Host: sftp.hostinger.com (or hostname from Hostinger)
   - Username: Your Hostinger user
   - Password: Your Hostinger password
   - Port: 22 (SFTP)

2. Navigate to: public_html/sentinel/

3. Drag files (or use synchronize feature for entire directory)

4. Upload order:
   - .env.example first
   - backend/ directory
   - pages/ directory (NEW)
   - assets/ directory
   - dashboard.php
   - Other PHP files
```

**Using Command Line (if SSH access available):**

```bash
# From your local machine, in sentinel project root
rsync -avz --exclude='.env' --exclude='vendor' \
  --exclude='node_modules' --exclude='.git' \
  ./ your-user@sftp.hostinger.com:~/public_html/sentinel/
```

**Files to EXCLUDE (Don't Upload):**
- `.env` (Never upload - create on server)
- `.git/` and `.gitignore`
- `vendor/` (Generate on server)
- `node_modules/`
- `*.log` files
- `agent_debug.log`
- `persistent-monitoring-agent/build/` (optional)

## Environment Configuration

### Step 1: Create .env File on Server

**Using Hostinger File Manager:**

1. Go to File Manager > sentinel/
2. Right-click > Create New File > `.env`
3. Double-click to edit
4. Copy content from `.env.example` and modify:

```env
# Database Configuration
DB_HOST=mysql.hostinger.com          # Or your Hostinger MySQL host
DB_USER=sentinel_user
DB_PASS=your_strong_password_here
DB_NAME=sentinel_prod
DB_PORT=3306

# API Configuration
SENTINEL_API_BASE=https://yourdomain.com/sentinel/backend/api/index.php
SENTINEL_API_KEY=your-secure-api-key-change-this
SENTINEL_ADMIN_KEY=your-admin-key-change-this
SENTINEL_BLOCK_REDIRECT_IP=127.0.0.2

# Optional: Email (configure later in Settings page)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_EMAIL=your-email@gmail.com
SMTP_PASSWORD=your-app-password

# Optional Settings
DEBUG=false
LOG_LEVEL=info
```

**Using SSH (if available):**

```bash
# SSH into Hostinger
ssh your-user@your-server.com

# Navigate to project
cd public_html/sentinel/

# Create .env from template
cp .env.example .env

# Edit .env with nano
nano .env

# Edit the values as shown above
# Press Ctrl+X, then Y, then Enter to save
```

### Step 2: Set File Permissions

```bash
# If using SSH
cd public_html/sentinel/

# Make directories writable
chmod 755 backend/
chmod 755 pages/
chmod 755 assets/
chmod 755 vendor/

# Make .env readable by PHP only
chmod 600 .env

# Make agent debug log writable (if needed)
chmod 755 .
```

## Composer Dependencies

Sentinel uses PHPMailer for SMTP email functionality.

### Step 1: Install Composer on Hostinger

Most Hostinger plans include Composer. Check by running:

```bash
# SSH into Hostinger
ssh your-user@your-server.com

# Check if composer is available
which composer

# If not found, check alternative commands:
command -v composer
/opt/alt/php80/usr/bin/composer  # Might be version-specific
```

### Step 2: Install Dependencies

```bash
# SSH into project directory
cd public_html/sentinel/

# Install dependencies from composer.json
composer install

# Or if composer is version-specific:
/opt/alt/php80/usr/bin/composer install

# Verify PHPMailer was installed
ls -la vendor/phpmailer/phpmailer/
```

### Step 3: If Composer Not Available

If your Hostinger plan doesn't include Composer:

**Option A: Use Pre-Built Vendor Package**

1. On your local machine:
```bash
composer install
tar -czf vendor.tar.gz vendor/
```

2. Upload vendor.tar.gz to server
3. SSH into server:
```bash
cd public_html/sentinel/
tar -xzf vendor.tar.gz
rm vendor.tar.gz
```

**Option B: Use Built-in Mail Function**

If you can't use PHPMailer, the system falls back to PHP's `mail()` function. This may work if Sendmail is configured on Hostinger.

## Troubleshooting

### Issue: 500 Internal Server Error

**Check PHP logs:**
```bash
cd public_html/sentinel/
tail -f ../logs/error_log

# Or check Hostinger error logs in File Manager
```

**Common causes:**
- `.env` file missing or invalid
- Database credentials wrong
- PHP version incompatibility (needs 7.4+)

### Issue: "Database Connection Failed"

**Check:**
1. PHP MySQL extension installed: `php -m | grep mysql`
2. Correct credentials in `.env`
3. Database actually created in phpMyAdmin
4. MySQL host correct (not localhost if using external DB)

```bash
# Test MySQL connection
mysql -h mysql.hostinger.com -u sentinel_user -p sentinel_prod -e "SELECT 1;"
```

### Issue: SMTP Email Not Sending

**Check:**
1. SMTP credentials correct
2. Firewall not blocking port 587/465
3. PHPMailer library loaded

```bash
# Verify vendor files
php -l vendor/autoload.php
php -r "require 'vendor/autoload.php'; echo 'OK';"
```

### Issue: Agent.py Connection Fails

**Verify:**
1. Agent has correct `SENTINEL_API_BASE` in .env
2. API endpoint is reachable:
   ```bash
   curl https://yourdomain.com/sentinel/backend/api/index.php?action=heartbeat
   ```
3. API key is correct

## Post-Deployment Maintenance

### Regular Tasks

**Weekly:**
- [ ] Check alert logs in dashboard
- [ ] Review active domains list
- [ ] Verify device heartbeats

**Monthly:**
- [ ] Backup database
  ```bash
  mysqldump -u sentinel_user -p sentinel_prod > backup.sql
  ```
- [ ] Archive old alerts
- [ ] Review audit logs
- [ ] Check agent version compatibility

**Quarterly:**
- [ ] Update PHP version (via Hostinger)
- [ ] Update dependencies:
  ```bash
  composer update
  ```
- [ ] Review and rotate SMTP password

### Backup Strategy

**Automated Hostinger Backups:**
1. Log in to Hostinger hPanel
2. Navigate to **Backups**
3. Check automatic backup frequency

**Manual Backup:**
```bash
# Backup files
tar -czf sentinel-files-$(date +%Y%m%d).tar.gz public_html/sentinel/

mysqldump -u sentinel_user -p sentinel_prod > sentinel-db-$(date +%Y%m%d).sql
```

**When new features are released:**

1. Download latest version
2. Backup current installation
3. Update only changed files via SFTP/File Manager

   ```bash

   mysql -u sentinel_user -p sentinel_prod < backend/database/new_migration.sql
   ```

5. Clear browser cache

## Security Hardening

### After Deployment

1. **Change Default Credentials:**
   ```sql
   UPDATE users SET password_hash = PASSWORD('new_strong_password') WHERE email = 'admin@sentinel.local';
   ```

2. **Restrict .env File:**
   - Ensure `.env` is not accessible via web browser
   - Add to `.htaccess`:
     ```apache
     <Files .env>
         deny from all
     </Files>
     ```

3. **Enable SSL:**
   - Hostinger usually auto-enables
   - Verify: `https://yourdomain.com/sentinel/` (has lock icon)

4. **Set Strong Permissions:**
   - Uploaded files: `644`
   - Directories: `755`
   - `.env`: `600`

5. **Regular Security Updates:**
   - Keep PHP updated
   - Update Composer dependencies monthly
   - Monitor for CVEs

## Performance Optimization (Optional)

If you notice slow performance:

1. **Enable PHP caching:**
   - Contact Hostinger to enable OPcache

2. **Database optimization:**
   ```sql
   -- SSH into server and run:
   ANALYZE TABLE devices;
   OPTIMIZE TABLE alerts;
   OPTIMIZE TABLE logs;
   ```

3. **Cleanup old data:**
   ```sql
   -- Archive alerts older than 90 days
   DELETE FROM alerts WHERE timestamp < DATE_SUB(NOW(), INTERVAL 90 DAY);
   DELETE FROM logs WHERE timestamp < DATE_SUB(NOW(), INTERVAL 90 DAY);
   ```
